# Demo Pack Selection

本文件给出当前最适合答辩手工走查的样本选择方案，目标是用最少样本覆盖最多演示点。

## 推荐先导出为 PDF 的 5 份简历

1. `C01_林泽远.md`
   - 用途: Candidate 高匹配后端样本
   - 适合展示: 上传、解析、推荐、投递、HR 约面

2. `C02_周清禾.md`
   - 用途: Candidate 高匹配前端样本
   - 适合展示: 推荐理由、技能匹配、前端方向解释

3. `C03_许逸晨.md`
   - 用途: 测试自动化方向样本
   - 适合展示: QA 岗位与自动化测试链路匹配

4. `C05_宋嘉锐.md`
   - 用途: 中匹配全栈样本
   - 适合展示: 有优势也有差距的 XAI 解释

5. `C07_顾安然.md`
   - 用途: 低匹配反例样本
   - 适合展示: 为什么不匹配，以及系统如何给出负向解释

## 推荐演示岗位组合

1. `J01` Kotlin Backend Engineer
2. `J02` Vue Frontend Engineer
3. `J03` QA Automation Engineer
4. `J06` Fullstack Engineer

## 推荐 Candidate 走查顺序

1. 用 `C01` 演示高匹配主路径。
2. 用 `C05` 演示中匹配解释。
3. 用 `C07` 演示低匹配或非技术方向反例。

## 推荐 HR 走查顺序

1. 先创建或导入 `J01`、`J02`、`J06`。
2. 用 `C01` 的投递做约面演示。
3. 用 `C07` 展示低匹配或淘汰说明。

## 下一步

1. 优先使用 `printable/C01_林泽远_resume_print.html` 导出第一份 PDF 样例，再按同版式扩展其他候选人。
2. 用 `multimodal/` 目录中的本地作品集与 GitHub 页面样例完成外链聚合演示。
3. 准备 1 个 HR 账号和 2 个 Candidate 账号做多角色演示。
